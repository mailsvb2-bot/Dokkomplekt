Meaning of this baseline
------------------------

A future build is allowed to replace it only if it preserves all user-facing behavior described in USER_BEHAVIOR_CONTRACT.md and passes REGRESSION_CONTOUR.md / REGRESSION_MATRIX.md checks.

Non-negotiable baseline rules
-----------------------------
1. Do not return bundled medical DOCX/DOCM templates.
2. Keep the project as a doctor-owned constructor: the doctor uploads their own Word templates.
3. Preserve block-03 custom button creation, rename and delete behavior.
4. Preserve popup/UI data priority over scanner data for generated documents.
5. Preserve patient subfolder naming according to the selected naming principle.
6. Preserve privacy and neutral medical wording constraints.
7. Do not remove existing user functionality while repairing regressions.
