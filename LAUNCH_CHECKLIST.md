# Launch checklist — production constructor

## P0 — блокеры релиза

- [ ] `python release_check.py` проходит локально.
- [ ] `python make_release_zip.py` создаёт чистый source-archive.
- [ ] GitHub Actions `Windows EXE Build` проходит на чистом runner.
- [ ] В артефактах есть `dist/MedicalDiaryAutofill.exe` и `release/MedicalDiaryAutofill_PRODUCTION_SOURCE.zip`.
- [ ] EXE запускается на чистой Windows-машине без Python/pip.
- [ ] В первом пользовательском сценарии нет встроенного медицинского комплекта шаблонов.
- [ ] Врач может выбрать несколько своих DOCX/DOCM-шаблонов.
- [ ] Программа распознаёт названия сверху листа и показывает подтверждение `Сделать кнопки с такими названиями?`.
- [ ] Кнопки появляются в блоке `03` с понятными названиями документов врача.
- [ ] При настройке программы появляется окно `Как называть сохранённую папку?` с галочками и предпросмотром.
- [ ] Обычное создание документов использует выбранное правило имени папки пациента.
- [ ] Пакетная обработка использует то же правило имени папки пациента.
- [ ] Проверены сценарии: загрузить первичный документ, отметить свои документы, создать без печати, создать и распечатать.
- [ ] Проверена печать на реальном Windows-принтере и сценарий `принтер не выбран / недоступен`.
- [ ] Проверены 20–50 обезличенных реальных DOCX-наборов разных врачей/специальностей.
- [ ] Windows Defender / SmartScreen проверены. Для платного трафика желательно подписать EXE code-signing сертификатом.

## P1 — коммерческий запуск

- [ ] Подготовлена страница/лендинг с честным описанием: программа локальная, документы не отправляются в облако.
- [ ] Подготовлена короткая инструкция: открыть EXE → добавить свои шаблоны → подтвердить кнопки → выбрать пациента → создать документы.
- [ ] Подготовлен способ поддержки: куда прислать обезличенный пример, лог/скрин ошибки.
- [ ] Подготовлена политика обработки данных: медицинские документы остаются у пользователя локально.
- [ ] Подготовлен rollback: предыдущий стабильный EXE и source-archive сохранены.

## P2 — после первых пользователей

- [ ] Собрать реальные ошибки на обезличенных примерах.
- [ ] Добавить регрессионные smoke-тесты на каждый повторяющийся сбой.
- [ ] Улучшать распознавание названий шаблонов по реальным документам, не добавляя зашитые медицинские формы.
