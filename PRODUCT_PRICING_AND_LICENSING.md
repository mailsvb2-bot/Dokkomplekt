# Dokkomplekt — продуктовая тарифная и лицензионная политика

## Продуктовая рамка

Dokkomplekt остаётся локальным Windows-конструктором медицинских документов из Word-шаблонов врача/отделения.

Нельзя менять базовый контракт продукта:

- документы пациента не загружаются наружу;
- телеметрия не добавляется;
- встроенные пользовательские медицинские шаблоны не поставляются;
- врач/клиника владеют своими DOCX/DOCM-шаблонами;
- врач подтверждает недостающие значения перед созданием документов;
- paid-версии не получают водяных знаков.

## Trial и водяные знаки

Trial создаёт документы только с footer-водяным знаком:

```text
ПРОБНАЯ ВЕРСИЯ. НЕ ИСПОЛЬЗОВАТЬ КАК МЕДИЦИНСКИЙ ДОКУМЕНТ.
```

Правила:

- paid active: без водяного знака;
- grace period: без водяного знака, но с предупреждением;
- expired/invalid license: рабочее создание документов заблокировано;
- expired demo mode: допускается только тестовый демо-документ с водяным знаком, если соответствующий UI-сценарий будет добавлен отдельно.

## Лицензии и доступы

Лицензия хранится локально в JSON-файле. Проверка лицензии не требует отправки документов пациента наружу.

Лицензионный payload содержит только продуктовые поля:

```json
{
  "license_id": "LIC-...",
  "plan": "doctor_pro",
  "owner_name": "...",
  "organization_name": "...",
  "seats": 2,
  "allowed_machines": ["hashed-machine-id"],
  "valid_until": "2027-06-27T00:00:00+00:00",
  "generation_limit_month": 3000,
  "template_limit": 150,
  "profile_limit": 3,
  "offline_grace_days": 7,
  "signature": "..."
}
```

Запрещено включать в лицензию:

- ФИО пациента;
- диагноз;
- тексты медицинских документов;
- названия пациентских файлов;
- содержимое шаблонов.

## Техническая реализация

- `product_licensing.py` — источник истины тарифов, trial, лимитов, machine binding, usage counters и offline-license проверки.
- `product_watermark.py` — post-processing водяного знака для trial/demo DOCX.
- `product_license_mixin.py` — локальный UI-диалог лицензии.
- `product_access_mixin.py` — обвязка создания документов лимитами, watermark-policy и счётчиком успешных DOCX.
- `tests/test_product_licensing_contract.py` — регрессионный контракт тарифов, trial, лимитов, grace и watermark-mode.

## Коммерческий принцип

Главные деньги проекта — не только подписка, а связка:

```text
лицензия + настройка шаблонов + отделенческий профиль + поддержка
```
