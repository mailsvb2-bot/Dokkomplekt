# Windows acceptance checklist — MedicalDiaryAutofill v1.4.53

Цель: проверить реальный врачебный сценарий на Windows, Word и Defender, который нельзя доказать в headless-среде.

## 1. Первый запуск

- [ ] Распаковать архив в чистую папку без кириллических спецсимволов в имени папки.
- [ ] Запустить `main.py` или EXE-сборку.
- [ ] Программа спросила про создание папки `Выписанные пациенты`.
- [ ] После согласия папка создана на рабочем столе.
- [ ] В блоке 04 нажать `Диагн.` — все ключевые строки понятны врачу.

## 2. Watcher при закрытой программе

- [ ] Закрыть программу полностью.
- [ ] Перенести первичный DOCX в папку `Выписанные пациенты`.
- [ ] PowerShell/cmd окно не появляется.
- [ ] Программа открывается сама.
- [ ] Окно выбора документов появляется один раз и не зависает.
- [ ] Повторный перенос того же файла не запускает бесконечный цикл.

## 3. Повторный запуск

- [ ] Закрыть программу.
- [ ] Открыть программу вручную.
- [ ] Снова закрыть программу.
- [ ] Перенести новый первичный DOCX в `Выписанные пациенты`.
- [ ] Программа снова открывается сама.

## 4. Документы и popup

- [ ] В блоке 03 нажать программно созданную кнопку документа.
- [ ] Если документу нужны номер истории/диагноз/лечение/дата/анализы/военкомат — popup появляется сразу.
- [ ] Военкомат можно ввести вручную.
- [ ] Введённый военкомат подтягивается в следующий popup.
- [ ] Popup анализов простой: нет анализов / вставить / сканер мышкой / сканер Word / файл.

## 5. Дневники

- [ ] Загрузить первичный DOCX с датой госпитализации.
- [ ] Программа подтягивает дату госпитализации.
- [ ] Программа автоподбирает тексты дневников по диагнозу, если папка текстов задана.
- [ ] Программа автоподбирает шаблон дат 01–31, если папка дат задана.
- [ ] При создании дневников появляется мастер дневников.
- [ ] Созданный DOCX содержит дату/месяц-год и тексты дневников.
- [ ] Если текстов нет — пустой дневник не создаётся.

## 6. Папка пациента

- [ ] Созданные документы сохраняются в уже созданную папку пациента внутри `Выписанные пациенты`.
- [ ] В папке пациента нет `_medical_autofill_history`.
- [ ] При повторном создании старый документ не теряется: создаётся backup/новая версия.

## 7. Окно программы

- [ ] Свернуть окно.
- [ ] Развернуть окно из панели задач.
- [ ] Окно возвращается и получает фокус.
- [ ] Горячие клавиши F5/F8/F9 не срабатывают поверх modal popup.

## 8. Что прислать при ошибке

- [ ] Скриншот окна `Диагн.`.
- [ ] Последние строки `desktop_intake_agent.log` из папки настроек.
- [ ] Название Windows и наличие Word/LibreOffice.
- [ ] Обезличенный первичный DOCX, на котором сценарий ломается.
